## 一、系统概述

### 1.1 产品定位
宠物百科全书内容后台管理系统，用于管理爬虫采集的宠物相关内容，通过人工审核确保内容质量，最终发布到前端百科。

### 1.2 核心原则
- **内容为王**：高质量内容是核心竞争力
- **爬虫为主**：90% 内容来自爬虫采集，10% 人工补充
- **人工审核**：所有爬虫内容必须经过人工审核才能发布
- **禁止 AI 生成**：严禁使用大模型生产内容，确保内容真实性和版权合规

### 1.3 目标用户
- 内容审核员：负责审核爬虫采集的内容
- 内容编辑：负责内容分类、标签管理、批量操作
- 系统管理员：负责爬虫配置、权限管理、系统监控

## 二、内容审核流程

### 2.1 状态流转图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   待审核    │────▶│   审核中    │────▶│   已通过    │
│  (Pending)  │     │ (Reviewing) │     │  (Approved) │
└─────────────┘     └─────────────┘     └─────────────┘
                         │
                         ▼
                   ┌─────────────┐
                   │   已拒绝    │
                   │  (Rejected) │
                   └─────────────┘
```

### 2.2 状态定义

| 状态 | 说明 | 触发条件 | 可执行操作 |
|------|------|----------|------------|
| 待审核 | 爬虫采集后初始状态 | 爬虫完成采集并清洗 | 开始审核、批量分配 |
| 审核中 | 审核员正在审核 | 审核员领取审核任务 | 通过、拒绝、退回 |
| 已通过 | 审核通过，可发布 | 审核员点击通过 | 发布、撤回 |
| 已拒绝 | 审核未通过 | 审核员点击拒绝 | 重新审核、删除 |

### 2.3 审核操作详情

#### 2.3.1 审核通过
- 内容进入"已通过"池
- 可立即发布或加入发布队列
- 记录审核员、审核时间、审核意见

#### 2.3.2 审核拒绝
- 必须填写拒绝原因（下拉选择 + 自定义输入）
- 拒绝原因分类：
  - 内容质量低（字数不足、图片模糊）
  - 内容重复（站内重复、全网重复）
  - 版权风险（来源不明、疑似侵权）
  - 内容违规（广告、敏感信息）
  - 其他
- 内容进入"已拒绝"池，可重新审核

#### 2.3.3 审核撤回
- 仅限"已通过"状态的内容
- 需填写撤回原因
- 内容返回"待审核"状态

### 2.4 审核任务分配

#### 2.4.1 自动分配
- 新内容按轮询方式分配给在线审核员
- 支持按审核员专长分配（如猫科、犬科、水族等）

#### 2.4.2 手动领取
- 审核员可从"待审核"池主动领取任务
- 每人同时审核中的任务上限：50 条

#### 2.4.3 审核时效
- 普通内容：24 小时内完成审核
- 紧急内容（用户举报、热门话题）：2 小时内完成审核
- 超时未审核：系统自动提醒，升级至主管

## 三、内容增删改查（CRUD）

### 3.1 内容列表页

#### 3.1.1 筛选条件
- 状态筛选：待审核、审核中、已通过、已拒绝、已发布
- 内容类型：品种百科、饲养指南、疾病知识、行为训练、营养美食
- 来源筛选：爬虫源 URL、爬取时间范围
- 审核员筛选：按审核员筛选
- 标签筛选：按标签筛选
- 关键词搜索：标题、内容全文搜索

#### 3.1.2 列表字段
| 字段 | 说明 |
|------|------|
| 标题 | 内容标题，点击可预览 |
| 状态 | 当前审核状态 |
| 类型 | 内容分类 |
| 来源 | 爬虫源 URL |
| 爬取时间 | 爬虫采集时间 |
| 审核员 | 当前审核员 |
| 创建时间 | 内容入库时间 |
| 操作 | 审核、编辑、删除等 |

### 3.2 内容详情页

#### 3.2.1 基本信息
- 标题、副标题
- 正文内容（支持富文本预览）
- 图片列表（支持放大查看）
- 标签列表
- 所属分类

#### 3.2.2 溯源信息
- 来源 URL（可点击跳转）
- 爬取时间
- 爬虫任务 ID
- 原始内容快照（保留采集时的原始 HTML）

#### 3.2.3 审核信息
- 当前状态
- 审核员
- 审核时间
- 审核意见
- 状态变更历史（时间线展示）

### 3.3 批量操作

#### 3.3.1 支持的批量操作
| 操作 | 适用状态 | 说明 |
|------|----------|------|
| 批量分配 | 待审核 | 批量分配给指定审核员 |
| 批量通过 | 待审核/审核中 | 批量审核通过（需谨慎使用） |
| 批量拒绝 | 待审核/审核中 | 批量拒绝，需选择拒绝原因 |
| 批量删除 | 已拒绝 | 批量删除已拒绝内容 |
| 批量发布 | 已通过 | 批量发布到前端 |
| 批量撤回 | 已发布 | 批量撤回内容 |
| 批量修改分类 | 所有状态 | 批量修改内容分类 |
| 批量添加标签 | 所有状态 | 批量添加标签 |

#### 3.3.2 批量操作限制
- 单次批量操作上限：100 条
- 批量通过/拒绝需二次确认
- 批量操作需记录操作日志

### 3.4 内容编辑

#### 3.4.1 可编辑字段
- 标题、副标题
- 正文内容（富文本编辑器）
- 图片（增删改、调整顺序）
- 标签（添加、删除）
- 分类（单选）

#### 3.4.2 编辑限制
- "审核中"状态：仅审核员可编辑
- "已通过/已发布"状态：编辑后自动退回"待审核"
- "已拒绝"状态：不可编辑，需重新审核

#### 3.4.3 编辑历史
- 记录每次编辑的时间、操作人、变更内容
- 支持版本对比和回滚

### 3.5 内容删除

#### 3.5.1 删除流程
- 软删除：内容标记为删除，实际保留在数据库
- 硬删除：永久删除（仅管理员可操作）

#### 3.5.2 删除限制
- "审核中"状态：不可删除，需先结束审核
- "已发布"状态：需先撤回再删除
- 批量删除仅限"已拒绝"状态

## 六、版权/合规管理

### 6.1 版权信息管理

#### 6.1.1 版权字段
| 字段 | 说明 |
|------|------|
| 版权声明 | 内容版权归属声明 |
| 原作者 | 原作者名称（如可追溯） |
| 原链接 | 原始内容链接 |
| 授权状态 | 已授权/未授权/待确认 |
| 授权类型 | CC 协议/付费授权/免费授权 |
| 授权凭证 | 授权书/邮件截图等 |

#### 6.1.2 版权审核流程
1. 爬虫采集时自动记录来源信息
2. 审核员审核时检查版权状态
3. 未授权内容需标记"待确认"
4. 高风险内容（商业网站、付费内容）需法务审核

### 6.2 合规检查

#### 6.2.1 违规内容类型
| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 广告推广 | 含联系方式、购买链接 | 拒绝 + 拉黑来源 |
| 敏感信息 | 政治、色情、暴力 | 拒绝 + 删除 + 记录 |
| 虚假信息 | 伪科学、谣言 | 拒绝 + 标记来源 |
| 侵权内容 | 明确侵权投诉 | 下架 + 法务处理 |
| 低质内容 | 字数不足、图片模糊 | 拒绝 |

#### 6.2.2 敏感词过滤
- 内置敏感词库（政治、色情、暴力、广告）
- 爬虫采集时自动过滤含敏感词内容
- 审核时高亮显示敏感词
- 支持自定义敏感词（按类别管理）

### 6.3 侵权投诉处理

#### 6.3.1 投诉渠道
- 前端页面侵权举报入口
- 邮箱投诉（copyright@petencyclopedia.com）
- 后台系统工单

#### 6.3.2 处理流程
1. 收到投诉，创建侵权工单
2. 核实投诉内容（24 小时内）
3. 确认侵权：下架内容 + 记录 + 回复投诉人
4. 不侵权：回复投诉人 + 恢复内容（如已下架）
5. 疑难案例：法务介入

#### 6.3.3 侵权记录
- 侵权内容 ID
- 投诉人信息
- 投诉时间
- 处理结果
- 处理人
- 侵权来源拉黑（如适用）

### 7.2 内容质量标准

#### 7.2.1 字数要求
| 内容类型 | 最低字数 | 建议字数 | 说明 |
|----------|----------|----------|------|
| 品种百科 | 800 字 | 1500-3000 字 | 需包含特征、历史、饲养要点 |
| 饲养指南 | 500 字 | 1000-2000 字 | 需包含具体操作步骤 |
| 疾病知识 | 600 字 | 1500-2500 字 | 需包含症状、治疗、预防 |
| 行为训练 | 400 字 | 800-1500 字 | 需包含训练步骤 |
| 营养美食 | 300 字 | 600-1200 字 | 需包含食谱、禁忌 |

#### 7.2.2 图片要求
| 要求 | 说明 |
|------|------|
| 最低数量 | 至少 1 张相关图片 |
| 建议数量 | 3-5 张（品种百科建议 5 张以上） |
| 图片质量 | 分辨率≥800x600，清晰无水印 |
| 图片格式 | JPG/PNG，单张≤2MB |
| 图片版权 | 优先使用 CC 协议图片，避免商业图片 |

#### 7.2.3 原创度要求
- **禁止 AI 生成**：严禁使用大模型生成内容
- **爬虫内容**：允许采集，但需人工审核 + 改写（避免直接复制）
- **改写标准**：核心信息保留，表达方式需调整，改写比例≥40%
- **原创内容**：鼓励人工原创，标记"原创"标签

#### 7.2.4 内容结构要求
- 标题清晰，不含夸张词汇
- 段落分明，每段≤200 字
- 含小标题（长内容）
- 含列表/表格（如适用）
- 无错别字、语病

### 7.3 去重规则

#### 7.3.1 内容指纹生成
- 标题指纹：标题 MD5（忽略标点、空格）
- 内容指纹：正文前 500 字 MD5
- 图片指纹：图片 URL MD5
- 综合指纹：标题 + 内容组合 MD5

#### 7.3.2 站内去重
| 相似度 | 处理方式 |
|--------|----------|
| ≥95% | 判定为重复，自动拒绝 |
| 80%-95% | 标记为疑似重复，人工审核判断 |
| <80% | 通过，允许入库 |

#### 7.3.3 全网去重
- 使用搜索引擎 API 检测全网重复
- 检测标题 + 前 200 字内容
- 发现高重复（≥80%）标记为"疑似采集"
- 高重复内容需人工审核，判断是否改写

#### 7.3.4 去重例外
- 同一来源的系列内容（如"XX 品种百科 - 上/中/下"）
- 内容更新（旧内容标记为"已更新"，新内容替换）
- 多语言内容（同一内容的不同语言版本）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  定时爬取   │────▶│   清洗      │────▶│   去重      │
│  (Crawler)  │     │  (Clean)    │     │ (Dedup)     │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                                              ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   发布      │◀────│  人工审核   │◀────│  待审核     │
│  (Publish)  │     │  (Review)   │     │ (Pending)   │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 8.2 定时爬取

#### 8.2.1 爬取计划配置
| 字段 | 说明 | 示例 |
|------|------|------|
| 任务名称 | 人类可读名称 | 波奇网品种百科爬取 |
| 爬虫源 | 关联的爬虫源 ID | SOURCE001 |
| 爬取频率 | 定时规则 | 每天 02:00 |
| 爬取深度 | 爬取层级 | 2 层 |
| 单次上限 | 单次最多采集数量 | 100 条 |
| 启用状态 | 是否启用 | 是 |

#### 8.2.2 爬取调度
- 支持 Cron 表达式配置
- 支持手动触发
- 支持暂停/恢复
- 爬取队列管理（优先级排序）

### 8.3 内容清洗

#### 8.3.1 清洗规则
| 清洗项 | 规则 |
|--------|------|
| HTML 标签 | 移除所有 HTML 标签，保留纯文本 |
| 广告内容 | 移除含广告关键词的段落 |
| 无关内容 | 移除"推荐阅读"、"相关推荐"等 |
| 联系方式 | 移除电话、微信、QQ 等 |
| 外链 | 移除所有外链（或替换为自有链接） |
| 格式整理 | 统一段落间距、标点符号 |

#### 8.3.2 图片处理
- 下载图片到本地存储
- 压缩图片（保持质量，减小体积）
- 生成缩略图
- 替换原文图片链接为本地链接

#### 8.3.3 清洗失败处理
- 清洗失败内容标记为"清洗失败"
- 记录失败原因
- 进入异常队列，人工处理
- 支持重新清洗

### 8.4 内容去重

#### 8.4.1 去重流程
1. 生成内容指纹（标题 + 内容）
2. 站内比对：与已有内容比对相似度
3. 全网比对：调用搜索引擎 API 检测
4. 判定结果：重复/疑似重复/通过
5. 重复内容：自动拒绝，记录原因
6. 疑似重复：进入待审核，标记"疑似重复"
7. 通过内容：进入待审核池

#### 8.4.2 去重配置
- 相似度阈值可配置（默认 95%）
- 支持白名单（允许重复的来源）
- 支持黑名单（强制去重的来源）

### 8.5 人工审核

#### 8.5.1 审核任务推送
- 新内容自动进入"待审核"池
- 支持实时推送（WebSocket）
- 支持邮件/短信通知（新任务达到阈值）

#### 8.5.2 审核界面
- 左侧：内容预览（标题、正文、图片）
- 右侧：审核操作（通过/拒绝/备注）
- 底部：溯源信息（来源 URL、爬取时间）

#### 8.5.3 审核辅助
- 敏感词高亮
- 重复内容提示
- 版权风险提示
- 快速备注（常用语）

### 8.6 内容发布

#### 8.6.1 发布方式
| 方式 | 说明 |
|------|------|
| 手动发布 | 审核员手动点击发布 |
| 批量发布 | 批量选择已通过内容发布 |
| 自动发布 | 定时自动发布（需配置） |

#### 8.6.2 发布队列
- 支持发布队列管理
- 支持定时发布（指定时间）
- 支持发布优先级（紧急内容优先）

#### 8.6.3 发布后处理
- 生成静态页面（如适用）
- 更新 sitemap
- 推送搜索引擎（百度、谷歌）
- 记录发布时间、发布人

### 8.7 异常处理

#### 8.7.1 爬取失败
| 异常类型 | 处理方式 |
|----------|----------|
| 网络超时 | 自动重试（最多 3 次），失败后记录日志 |
| 网站结构变更 | 标记任务失败，通知管理员更新爬虫规则 |
| 反爬限制 | 暂停爬取，切换代理 IP，通知管理员 |
| 内容解析失败 | 记录失败内容，进入异常队列 |

#### 8.7.2 内容违规
| 违规类型 | 处理方式 |
|----------|----------|
| 敏感内容 | 自动拒绝，记录来源，加入黑名单 |
| 侵权内容 | 下架内容，记录侵权信息，法务处理 |
| 广告内容 | 拒绝内容，降低来源质量评分 |
| 低质内容 | 拒绝内容，连续低质则禁用来源 |

#### 8.7.3 系统异常
| 异常类型 | 处理方式 |
|----------|----------|
| 数据库异常 | 自动重试，失败后告警 |
| 存储异常 | 切换备用存储，告警 |
| 服务宕机 | 自动重启，告警通知 |

#### 8.7.4 告警通知
- 告警级别：P0（紧急）/P1（高）/P2（中）/P3（低）
- 通知渠道：邮件/短信/飞书
- 告警升级：P0/P1 级别 30 分钟未处理自动升级

## 十三、风险与应对

### 13.1 版权风险
- **风险**：爬虫内容可能涉及侵权
- **应对**：人工审核 + 版权信息记录 + 侵权投诉快速处理

### 13.2 内容质量风险
- **风险**：爬虫内容质量参差不齐
- **应对**：严格审核标准 + 来源质量评分 + 低质来源淘汰

### 13.3 技术风险
- **风险**：反爬机制导致采集失败
- **应对**：多代理 IP + 爬取频率控制 + 人工补充内容

### 13.4 合规风险
- **风险**：内容含敏感信息
- **应对**：敏感词过滤 + 人工审核 + 违规内容快速下架
