### 任务 1：修复 W1/W2 遗留的构建问题

**问题 1：front-matter 模块导入错误**
- **错误信息：** `Module '"front-matter"' has no exported member 'parse'`
- **根本原因：** front-matter 使用默认导出，而非命名导出
- **修复方案：** 将 `import { parse } from 'front-matter'` 改为 `import parse from 'front-matter'`
- **影响文件：** `src/scripts/import-content.ts`

**问题 2：API 路由动态渲染配置**
- **错误信息：** `Dynamic server usage: Route /api/v1/categories couldn't be rendered statically`
- **根本原因：** Next.js 14 默认尝试静态生成，但 API 路由使用了 `request.url`
- **修复方案：** 为所有 API 路由添加 `export const dynamic = 'force-dynamic'`
- **影响文件：** 
  - `src/app/api/v1/categories/route.ts`
  - `src/app/api/v1/articles/route.ts`
  - `src/app/api/v1/search/route.ts`
  - `src/app/api/v1/articles/[slug]/route.ts`

**问题 3：搜索页面 Suspense 边界**
- **错误信息：** `useSearchParams() should be wrapped in a suspense boundary`
- **根本原因：** Next.js 要求使用 `useSearchParams` 的组件必须有 Suspense 边界
- **修复方案：** 将搜索内容逻辑提取到独立组件，并用 Suspense 包装
- **影响文件：** `src/app/(public)/search/page.tsx`

**构建结果：** ✅ 成功通过，无错误无警告

### 任务 3：内容导入脚本完善

**问题发现**
- tsx 运行脚本时无法自动加载 `.env.local` 文件
- 导致 `process.env.DATABASE_URL` 为 undefined

**解决方案**
1. 安装 dotenv：`npm install dotenv`
2. 在脚本中手动加载：
```typescript
import * as dotenv from 'dotenv';
import { fileURLToPath } from 'url';
import { dirname } from 'path';

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);
dotenv.config({ path: join(__dirname, '../../.env.local') });
```
