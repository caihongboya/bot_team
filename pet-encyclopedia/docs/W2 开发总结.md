## 任务 1：搜索功能优化（中文分词改进）

### 当前问题
W1 阶段使用 PostgreSQL 的 `simple` 配置进行全文搜索，仅按字符分割，中文搜索效果不佳。

### 解决方案

#### 方案 A：pg_jieba 中文分词扩展（推荐）
- 安装 PostgreSQL 的 jieba 分词扩展
- 修改搜索向量生成函数使用 jieba 分词
- 优点：搜索准确度高，支持词组匹配
- 缺点：需要安装扩展，部署稍复杂

#### 方案 B：简单优化（MVP 备选）
- 保持 simple 配置
- 优化查询逻辑，支持关键词组合
- 优点：零额外依赖
- 缺点：搜索体验一般

### 实施步骤
1. 评估 pg_jieba 安装可行性
2. 修改 schema.sql 中的搜索向量生成函数
3. 更新搜索 API 使用新的分词配置
4. 测试搜索效果

## W3 详细开发记录

### 2026-03-17 14:45 - 任务 1：修复 W1/W2 遗留的构建问题

**问题 1：front-matter 模块导入错误**
- 错误：`Module '"front-matter"' has no exported member 'parse'`
- 修复：将 `import { parse } from 'front-matter'` 改为 `import parse from 'front-matter'`
- 文件：`src/scripts/import-content.ts`

**问题 2：API 路由动态渲染配置**
- 错误：API 路由使用了 `request.url` 导致无法静态生成
- 修复：为所有 API 路由添加 `export const dynamic = 'force-dynamic'`
- 文件：`src/app/api/v1/categories/route.ts`, `src/app/api/v1/articles/route.ts`, `src/app/api/v1/search/route.ts`, `src/app/api/v1/articles/[slug]/route.ts`

**问题 3：搜索页面 Suspense 边界**
- 错误：`useSearchParams() should be wrapped in a suspense boundary`
- 修复：将搜索内容包装在 Suspense 组件中
- 文件：`src/app/(public)/search/page.tsx`
