## 2. 环境搭建

### 2.1 安装 Node.js

#### macOS
```bash
# 使用 Homebrew
brew install node@20

# 或使用 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
nvm use 20
```

#### Windows
1. 访问 https://nodejs.org/
2. 下载 LTS 版本（20.x）
3. 运行安装程序

#### Linux (Ubuntu/Debian)
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 2.2 安装 PostgreSQL

#### macOS
```bash
brew install postgresql@15
brew services start postgresql@15
```

#### Windows
1. 访问 https://www.postgresql.org/download/windows/
2. 下载并安装 PostgreSQL 15
3. 记录安装时设置的密码

#### Linux (Ubuntu/Debian)
```bash
sudo apt-get update
sudo apt-get install -y postgresql-15 postgresql-contrib-15
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### 2.3 创建数据库

```bash
# 登录 PostgreSQL
psql -U postgres

# 创建数据库和用户
CREATE DATABASE pet_encyclopedia;
CREATE USER pet_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE pet_encyclopedia TO pet_user;

# 退出
\q
```

## 3. 项目初始化

### 3.1 克隆项目
```bash
cd /home/kite/.openclaw/workspace/pet-encyclopedia
```

### 3.2 安装依赖
```bash
npm install
```

### 3.3 配置环境变量

创建 `.env.local` 文件：

```bash
# 数据库配置
DATABASE_URL=postgresql://pet_user:your_password@localhost:5432/pet_encyclopedia

# 应用配置
NEXT_PUBLIC_SITE_URL=http://localhost:3000
NEXT_PUBLIC_API_URL=/api/v1

# 对象存储（可选，MVP 阶段可用本地存储）
# OSS_ACCESS_KEY_ID=your_access_key
# OSS_ACCESS_KEY_SECRET=your_secret_key
# OSS_BUCKET=pet-encyclopedia
# OSS_ENDPOINT=oss-cn-shanghai.aliyuncs.com
```

### 3.4 初始化数据库

```bash
# 运行 schema 脚本
npm run db:init

# 或手动执行
psql -U pet_user -d pet_encyclopedia -f src/lib/schema.sql
```

### 3.5 导入示例内容

```bash
# 导入所有示例内容
npm run content:import

# 或导入指定目录
npm run content:import -- --path src/content/cats
```

## 7. 常见问题

### 7.1 数据库连接失败

**错误信息**: `Error: connect ECONNREFUSED`

**解决方案**:
1. 检查 PostgreSQL 服务是否启动
2. 确认 DATABASE_URL 配置正确
3. 检查防火墙设置

```bash
# macOS 检查服务状态
brew services list

# Linux 检查服务状态
sudo systemctl status postgresql
```

### 7.2 端口被占用

**错误信息**: `Error: listen EADDRINUSE: address already in use :::3000`

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或修改端口
PORT=3001 npm run dev
```

### 7.3 内容导入失败

**错误信息**: `Category not found: xxx`

**解决方案**:
1. 确认数据库已正确初始化（运行 `npm run db:init`）
2. 检查 Markdown 文件的 frontmatter 中 category 和 subcategory 字段是否正确
3. 确认分类 slug 与数据库中一致

### 7.4 TypeScript 类型错误

**解决方案**:
```bash
# 清除缓存重新编译
rm -rf .next
npm run dev
```
