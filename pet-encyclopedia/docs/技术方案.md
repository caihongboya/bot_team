## 1. 技术选型

### 1.1 技术栈总览

| 层级 | 技术选择 | 理由 |
|------|----------|------|
| **前端框架** | Next.js 14 + React | SSR/SSG 支持，SEO 友好，生态成熟 |
| **后端服务** | Next.js API Routes | 轻量级，与前端同构，减少部署复杂度 |
| **数据库** | PostgreSQL 15+ | 成熟稳定，支持全文搜索，成本低 |
| **搜索方案** | PostgreSQL 全文搜索 (tsvector) | MVP 阶段足够，后续可升级 Meilisearch |
| **内容存储** | Markdown 文件 + 数据库元数据 | 便于版本管理，支持协作编辑 |
| **图片存储** | 阿里云 OSS / AWS S3 | CDN 加速，成本低 |
| **部署平台** | Vercel (前端) + Supabase (数据库) | 快速部署，免费额度足够 MVP |
| **版本控制** | Git + GitHub | 标准流程，支持 CI/CD |

### 1.2 技术栈详细说明

#### 前端框架：Next.js 14
- **App Router**：使用最新的 App Router 架构
- **Server Components**：服务端渲染，提升首屏性能
- **TypeScript**：类型安全，减少运行时错误
- **Tailwind CSS**：快速样式开发，响应式设计

#### 数据库：PostgreSQL
- **全文搜索**：使用 tsvector/tsquery 实现中文分词搜索
- **JSONB 支持**：灵活存储元数据
- **索引优化**：支持 GIN 索引加速搜索

#### 搜索方案对比

| 方案 | 适用阶段 | 优势 | 劣势 |
|------|----------|------|------|
| PostgreSQL 全文搜索 | MVP (≤500 篇) | 零额外部署，维护成本低 | 中文分词需配置，性能有限 |
| Meilisearch | 扩展阶段 (500-5000 篇) | 搜索体验好，支持拼音/纠错 | 需额外部署，增加运维成本 |
| Elasticsearch | 大规模 (5000+ 篇) | 功能强大，可扩展 | 资源消耗大，运维复杂 |

**MVP 决策：PostgreSQL 全文搜索**

### 3.2 表结构定义

#### 3.2.1 分类表 (categories)

```sql
CREATE TABLE categories (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,          -- 分类名称
  slug VARCHAR(100) NOT NULL UNIQUE,   -- URL 友好的标识
  parent_id INTEGER REFERENCES categories(id), -- 父分类 ID
  icon VARCHAR(50),                     -- 图标（emoji 或 icon 名称）
  sort_order INTEGER DEFAULT 0,         -- 排序顺序
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_categories_parent ON categories(parent_id);
CREATE INDEX idx_categories_slug ON categories(slug);
```

#### 3.2.2 文章表 (articles)

```sql
CREATE TABLE articles (
  id SERIAL PRIMARY KEY,
  title VARCHAR(200) NOT NULL,          -- 文章标题
  slug VARCHAR(200) NOT NULL UNIQUE,    -- URL 友好的标识
  category_id INTEGER NOT NULL REFERENCES categories(id),
  subcategory_id INTEGER REFERENCES categories(id),
  summary TEXT,                         -- 摘要
  content_md TEXT NOT NULL,             -- Markdown 原文
  content_html TEXT,                    -- 渲染后的 HTML
  cover_image VARCHAR(500),             -- 封面图 URL
  status VARCHAR(20) DEFAULT 'draft',   -- draft/published/archived
  view_count INTEGER DEFAULT 0,         -- 浏览量
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_articles_category ON articles(category_id);
CREATE INDEX idx_articles_status ON articles(status);
CREATE INDEX idx_articles_slug ON articles(slug);
CREATE INDEX idx_articles_created ON articles(created_at DESC);
```

#### 3.2.3 标签表 (tags)

```sql
CREATE TABLE tags (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50) NOT NULL UNIQUE,
  slug VARCHAR(50) NOT NULL UNIQUE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 文章标签关联表
CREATE TABLE article_tags (
  article_id INTEGER REFERENCES articles(id) ON DELETE CASCADE,
  tag_id INTEGER REFERENCES tags(id) ON DELETE CASCADE,
  PRIMARY KEY (article_id, tag_id)
);

-- 索引
CREATE INDEX idx_article_tags_article ON article_tags(article_id);
CREATE INDEX idx_article_tags_tag ON article_tags(tag_id);
```

#### 3.2.4 搜索索引表 (article_search)

```sql
-- 使用 PostgreSQL 全文搜索
CREATE TABLE article_search (
  article_id INTEGER PRIMARY KEY REFERENCES articles(id) ON DELETE CASCADE,
  search_vector tsvector NOT NULL
);

-- GIN 索引加速搜索
CREATE INDEX idx_article_search_vector ON article_search USING GIN(search_vector);

-- 触发器：自动更新搜索索引
CREATE OR REPLACE FUNCTION update_article_search_vector()
RETURNS trigger AS $$
BEGIN
  INSERT INTO article_search (article_id, search_vector)
  VALUES (
    NEW.id,
    setweight(to_tsvector('simple', NEW.title), 'A') ||
    setweight(to_tsvector('simple', COALESCE(NEW.summary, '')), 'B') ||
    setweight(to_tsvector('simple', strip_tags(COALESCE(NEW.content_html, ''))), 'C')
  )
  ON CONFLICT (article_id) DO UPDATE SET
    search_vector = EXCLUDED.search_vector;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_article_search_vector
  AFTER INSERT OR UPDATE ON articles
  FOR EACH ROW
  EXECUTE FUNCTION update_article_search_vector();
```

## 6. 搜索方案

### 6.1 PostgreSQL 全文搜索实现

#### 6.1.1 中文分词配置

PostgreSQL 默认不支持中文分词，需使用以下方案：

**方案 A：simple 配置 + 应用层分词**
```sql
-- 使用 simple 配置（按空格分词）
-- 在应用层对中文内容进行处理
search_vector = to_tsvector('simple', processed_text)
```

**方案 B：pg_jieba 扩展（推荐）**
```sql
-- 安装 pg_jieba 扩展
CREATE EXTENSION IF NOT EXISTS pg_jieba;

-- 使用中文分词
search_vector = to_tsvector('jiebacfg', content)
```

**MVP 决策：方案 A（simple 配置）**
- 无需额外安装扩展
- 在应用层对标题、摘要、内容进行预处理
- 后续可升级到 pg_jieba 或 Meilisearch

#### 6.1.2 搜索查询实现

```typescript
// 搜索服务实现
async function searchArticles(query: string, page: number, limit: number) {
  const searchQuery = query
    .split('') // 简单字符分割
    .filter(c => c.trim())
    .join(' | '); // OR 连接

  const sql = `
    SELECT 
      a.id,
      a.title,
      a.slug,
      a.summary,
      a.cover_image,
      c.name as category_name,
      c.slug as category_slug,
      ts_rank(as.search_vector, websearch_to_tsquery('simple', $1)) as score
    FROM articles a
    JOIN categories c ON a.category_id = c.id
    JOIN article_search as ON a.id = as.article_id
    WHERE as.search_vector @@ websearch_to_tsquery('simple', $1)
      AND a.status = 'published'
    ORDER BY score DESC
    LIMIT $2 OFFSET $3
  `;

  const results = await db.query(sql, [query, limit, (page - 1) * limit]);
  return results;
}
```

### 6.2 搜索优化策略

| 优化项 | 实现方式 |
|--------|----------|
| **权重分配** | 标题权重 A > 摘要权重 B > 内容权重 C |
| **相关度排序** | 使用 ts_rank() 函数 |
| **关键词高亮** | 使用 ts_headline() 函数 |
| **搜索建议** | 记录热门搜索词，提供自动补全 |
| **拼写纠错** | 后续集成 Meilisearch 实现 |

### 9.2 环境变量配置

```bash
# 数据库

OSS_ACCESS_KEY_SECRET=xxx

```

### 9.3 CI/CD 流程

```yaml
# .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Install dependencies
        run: npm ci
      - name: Build
        run: npm run build
      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

## 10. 风险评估与应对

### 10.1 技术风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| PostgreSQL 中文搜索效果不佳 | 中 | 中 | 预留 Meilisearch 升级方案，W3 评估效果 |
| 内容导入工具开发延期 | 低 | 中 | 简化 MVP 版本，支持手动导入 |
| Vercel 国内访问慢 | 中 | 高 | 准备国内 CDN 备选方案（阿里云 CDN） |
| SEO 效果不达标 | 中 | 中 | 优化 SSR、sitemap、结构化数据 |

### 10.2 项目风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 内容团队交付延期 | 中 | 高 | 提前准备 50 篇示例内容，技术侧可演示 |
| 需求变更 | 中 | 中 | 冻结 MVP 需求，变更放入 V2 |
| 人员协作问题 | 低 | 中 | 每日站会同步进度，飞书文档留痕 |

### 10.3 应急预案

**预案 A：搜索效果不达标**
- 触发条件：W3 测试搜索准确率 < 80%
- 应对：快速集成 Meilisearch（1-2 天）

**预案 B：内容交付不足**
- 触发条件：W3 结束内容 < 100 篇
- 应对：技术侧补充基础内容，保证 MVP 上线

**预案 C：部署延期**
- 触发条件：W4 第 3 天未完成部署
- 应对：先上线内测版，收集反馈迭代
